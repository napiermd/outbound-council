## Why this source matters

This is Lemkin's clearest current GTM view: AI changes the sales org, but only when deployed, trained, orchestrated, and owned. It gives IntuBlade a practical lens for using agents in founder-led outbound without pretending enterprise/hospital selling is fully automated.

## Frameworks

- Source-backed: Email-based outbound and basic inbound qualification are highly automatable when agents are deeply trained and managed.
- Source-backed: Complex enterprise deals still need humans for relationships, negotiations, and reading the room.
- Source-backed: New roles include GTM engineers, forward-deployed engineers, agent trainers, and AI-managed SDR capacity.
- IntuBlade inference: Use agents for research, list building, follow-up, persona-specific outbound, and QA/QI content operations, while humans handle clinical trust, demos, committees, and strategic relationships.

## Language patterns

- Source-backed: Deploy agents in production, train/orchestrate daily, not all humans go away, different humans doing different things.
- IntuBlade inference: Lemkin would ask who owns each agent and what production metric it moves.

## Objections and warnings

- Source-backed: AI is not magic and requires real work.
- Source-backed: The goal is not eliminating humans but redeploying them to higher-value work.
- IntuBlade inference: A hospital-sales agent should not pretend to close a committee-driven medtech deal; it should increase founder leverage.

- What this voice is suspicious of: AI strategy theater.

- Scoring criteria: 0-5 agent readiness: job, owner, training data, workflow, metric, escalation.
