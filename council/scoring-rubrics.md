# Scoring Rubrics

Use 1-10 scores. Scores below 7 require revision. Scores below 5 mean the artifact is probably pointed at the wrong buyer, category, or decision.

## EMS outbound email

| Criterion | What good looks like |
| --- | --- |
| Signal specificity | Opens with a real, current, verified detail from the prospect's world. |
| Buyer change trigger | Names a reason to rethink airway coverage now. |
| Category clarity | Frames IntuBlade as field-ready video airway coverage, not just a cheaper scope. |
| Trust | Uses physician-founder credibility and peer proof without chest-thumping. |
| Risk removal | Makes the deployment kit feel easy, safe, and worthwhile. |
| Reply path | Ends with a concrete operational question. |
| Human voice | Passes anti-AI-slop rules and sounds like Andrew or the rep. |

## Hospital sales package

| Criterion | What good looks like |
| --- | --- |
| Clinical need | Clear problem, stakeholder, and patient/workflow relevance. |
| Economic case | Cost comparison and value are specific, not hand-wavy. |
| Evidence package | Claims are supported and appropriately bounded. |
| Value-analysis readiness | Supply chain, risk, infection prevention, training, and finance have answers. |
| Champion strategy | Clinical and economic champions are identified and enabled. |
| Pilot design | Pilot has success criteria, timeline, owners, and next decision. |
| No-decision reduction | Makes choosing feel safer than waiting. |

## Founder-led narrative

| Criterion | What good looks like |
| --- | --- |
| Old game/new game | The market shift is clear and relevant. |
| Category POV | Buyer can understand what IntuBlade is and what it replaces. |
| Founder authority | Andrew's background earns trust without becoming the whole story. |
| Proof sequence | Claims escalate from observed problem to product proof to adoption proof. |
| Buyer relevance | Story starts in the buyer's world. |
| Repeatability | Narrative can survive repeated emails, calls, decks, and conference conversations. |

## Procedural-guidance AI

| Criterion | What good looks like |
| --- | --- |
| Workflow fit | Fits clinician behavior under real pressure. |
| Safety boundary | Does not overclaim or imply unsupported autonomy. |
| Evidence threshold | Names what must be proven before clinical adoption. |
| Buyer clarity | Distinguishes buyer, user, approver, and risk owner. |
| Hardware/software bridge | Shows why IntuBlade's device footprint creates a software advantage. |
| Adoption path | Starts with a credible pilot, not a moonshot claim. |

## Buyer-language review

| Criterion | What good looks like |
| --- | --- |
| Role fit | Sounds like the stakeholder's actual job, constraints, and vocabulary. |
| Trigger clarity | Names a situation that would make this buyer care now. |
| Proof match | Asks for or offers the proof this buyer needs. |
| Operational realism | Respects workflow, budget, training, and committee burden. |
| Inference discipline | Separates direct buyer language from council guesses. |

## Clinical-airway safety review

| Criterion | What good looks like |
| --- | --- |
| Procedural realism | Accounts for positioning, oxygenation, suction, tube delivery, and stress. |
| Evidence boundary | Does not imply unsupported outcomes or autonomy. |
| Failure modes | Names contamination, lens failure, physiology, and backup plans when relevant. |
| Training fit | Includes competency, deliberate practice, simulation, and debrief needs. |
| Team workflow | Addresses EMS/ED/anesthesia/RT roles instead of only the intubator. |

## Competitor-positioning review

| Criterion | What good looks like |
| --- | --- |
| Competitor appeal | Accurately states why the buyer might prefer the incumbent or substitute. |
| IntuBlade wedge | Names a specific gap IntuBlade can own. |
| Claim safety | Avoids unsupported superiority, safety, infection, cost, or availability claims. |
| Buyer relevance | Frames comparison around the buyer's workflow and adoption gate. |
| Proof path | States what evidence would make the comparison stronger. |

## Economic-case review

| Criterion | What good looks like |
| --- | --- |
| Comparator clarity | Defines the current workflow/product/economic baseline. |
| Budget-impact logic | Names cost drivers and assumptions without pretending they are proven. |
| Value-analysis readiness | Includes clinical sponsor, economic sponsor, evidence, risk, and pilot criteria. |
| Reimbursement discipline | Does not imply coding, coverage, or payment without support. |
| Adoption gate | States the next decision: trial, VAC, contract, stocking, scale, or no-go. |
