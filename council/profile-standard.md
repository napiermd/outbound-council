# Profile Standard

Every GTM council profile should be source-backed, not vibe-backed.

Use this file with:

- `source-index.md`
- `refresh-checklist.md`
- `roster.md`

## Required sections

Each `references/*.md` voice file should include:

1. `Core domain`
2. `Why this voice matters in this council`
3. `Canonical sources`
4. `Current signals`
5. `What this voice cares about`
6. `Signature questions`
7. `What this voice respects`
8. `What this voice is suspicious of`
9. `What this voice would push on`
10. `Common advice pattern`
11. `Best paired with`
12. `Tension with other voices`
13. `Decision style`
14. `Core beliefs / worldview`
15. `Likely next move (inference)`
16. `Watch sources`
17. `IntuBlade applications`
18. `Source mining backlog`
19. `Voice extraction targets`
20. `Last reviewed`

## Source rules

- Prefer first-party sources: official websites, books, newsletters, podcasts, conference talks, company blogs, and authored posts.
- Use secondary summaries only when they add current context that is not visible in first-party sources.
- Separate sourced positions from persona inference.
- Do not treat a famous name as a voice unless there is enough material to emulate their decision style.
- Mark source-thin profiles as `partially source-backed`.

## Current signals

`Current signals` should answer:

- what they appear to be working on now
- what role or platform they currently hold
- what themes are prominent in their recent public output

When a profile is older than 90 days, refresh it before relying on current signals.

## Enrichment sections

`IntuBlade applications` should translate the voice into actual use cases:

- disposable video laryngoscopy
- EMS and fire outreach
- hospital/value-analysis entry
- QA/QI software
- computer vision and procedural guidance
- founder-led GTM

`Source mining backlog` should name the next sources to mine. Good backlog items include:

- specific books or chapters
- specific podcast feeds or episodes
- first-party essays or newsletters
- talks, conference pages, or interviews
- social profiles only when they contain substantive source material

`Voice extraction targets` should say what to pull from those sources:

- frameworks
- stories and examples
- recurring phrases or distinctions
- objections and warnings
- decision heuristics
- draft-review scoring criteria

## Last reviewed

Every profile should end with:

- review date
- source count
- source confidence: `fully source-backed`, `mostly source-backed`, or `partially source-backed`
