# Field Signal Feedback Loop

Use this folder to turn real market signal into council improvements.

## Safety rules

- Do not store PHI.
- Scrub patient details, dates of service, names, MRNs, incident addresses, and identifiable clinical facts.
- Scrub customer identifiers unless intentionally preserved for account work.
- Label direct buyer language separately from inference.
- Mark confidence: `direct quote`, `paraphrase`, `rep inference`, or `council inference`.

## Weekly loop

1. Add new replies, call notes, demo outcomes, objections, and win/loss learnings.
2. Tag each signal by buyer archetype, pack, source, and stage.
3. Update the objection bank and buyer archetypes when patterns repeat.
4. Update council profiles only when a field pattern changes how a voice should behave.
5. Review one outbound, one deal, and one competitor objection using the updated signal.
